# 管理报表分析

使用本技能时，你需要为企业**编制管理类报表**，包括**部门损益表、项目利润表、KPI仪表板、经营分析报告**，为管理层决策提供数据支持。

---

## 一、管理报表概述

### 1.1 财务会计 vs 管理会计报表

| 维度 | 财务会计报表 | 管理会计报表 |
|------|-------------|-------------|
| 目标用户 | 外部（税务/银行/投资者） | 内部（管理层/部门负责人） |
| 准则要求 | 会计准则（CAS/IFRS/GAAP） | 无强制准则，按需设计 |
| 报告频率 | 月度/季度/年度 | 可按周/日/实时 |
| 颗粒度 | 公司层面汇总 | 部门/项目/产品维度 |
| 关注重点 | 合规、准确性 | 决策、控制、绩效 |

### 1.2 管理报表的核心原则
- **相关性**：信息与决策需求匹配
- **及时性**：快速出数（通常T+3以内）
- **可追溯性**：管理数据与财务数据可勾稽
- **可行动性**：每个数据点都能引导管理行动

## 二、部门损益表

### 2.1 部门损益表结构

```
营业收入（部门直接收入）
- 变动成本
  - 直接材料
  - 直接人工（计件）
  - 变动制造费用
= 边际贡献
- 部门可控固定成本
  - 部门管理人员工资
  - 部门折旧
  - 部门办公费用
= 部门可控利润
- 部门不可控固定成本（总部分摊）
  - 总部管理费用分摊
  - IT系统分摊
  - 行政后勤分摊
= 部门净利润
```

### 2.2 部门收入确认规则
- **直接收入**：部门独立产生的收入（如销售部的产品销售收入）
- **内部转移收入**：部门间服务/产品的内部结算价格
- **分摊收入**：多部门协同产生的收入按规则分配

### 2.3 部门成本分摊方法
| 分摊类型 | 分摊基础 | 适用场景 |
|---------|---------|---------|
| 按人头 | 部门人数占比 | 行政、HR、IT支持 |
| 按面积 | 使用面积占比 | 租金、物业、水电 |
| 按收入 | 收入占比 | 总部营销投入 |
| 按工时 | 服务工时占比 | IT、财务、法务 |
| 按设备 | 设备数量/价值 | IT设备折旧、软件许可 |

### 2.4 部门考核指标
- **边际贡献率** = 边际贡献 / 收入 × 100%
- **可控利润率** = 部门可控利润 / 收入 × 100%
- **人均创收** = 部门收入 / 部门人数
- **人均创利** = 部门可控利润 / 部门人数
- **费用率** = 部门费用 / 部门收入 × 100%

## 三、项目利润表

### 3.1 项目损益表结构

```
项目收入（合同金额/里程碑确认收入）
- 项目直接成本
  - 人工成本（项目人员工时 × 费率）
  - 外包成本
  - 差旅费用
  - 材料/设备成本
= 项目毛利
- 项目间接费用分摊
  - 项目管理费
  - 质量成本
  - 资金成本（占用资金利息）
= 项目净利润
```

### 3.2 项目盈利跟踪指标
| 指标 | 公式 | 目标参考 |
|------|------|---------|
| 项目毛利率 | 项目毛利/收入 | 视行业，通常 > 30% |
| 项目净利率 | 项目净利润/收入 | 视行业，通常 > 10% |
| 预算偏差率 | (实际成本-预算成本)/预算成本 | < ±5% 为良好 |
| 项目ROI | 项目净利润/项目投入成本 | > 20% 为优秀 |
| 人工利用率 | 实际收费工时/可计费工时 | > 75% |

### 3.3 项目全生命周期利润跟踪
```
阶段1：投标 → 预计利润率评估
阶段2：启动 → 初始预算利润
阶段3：执行 → 月度滚动利润预测
阶段4：收尾 → 最终结算利润率
阶段5：复盘 → 与实际对比分析偏差原因
```

## 四、KPI仪表板

### 4.1 财务类KPI
| 维度 | KPI | 频率 | 预警线 |
|------|-----|------|--------|
| 收入 | 累计收入 vs 预算完成率 | 周 | < 80% |
| 利润 | 净利润率 | 月 | < 5% |
| 现金 | 自由现金流 | 周 | < 0 |
| 回款 | DSO(应收账款周转天数) | 月 | > 60天 |
| 成本 | 成本收入比 | 月 | > 85% |

### 4.2 运营类KPI
| KPI | 公式 | 行业参考 |
|-----|------|---------|
| 客户获取成本(CAC) | 销售+营销费用/新增客户数 | 因行业而异 |
| 客户生命周期价值(LTV) | 平均客单价×复购次数×毛利率 | LTV/CAC > 3 |
| 库存周转天数 | 平均库存/日均成本 | 视行业 |
| 订单履约率 | 按时交付订单/总订单 | > 95% |
| 客户满意度(CSAT) | 满意评分/总评分 | > 85% |

### 4.3 KPI仪表板设计原则
- **少于10项KPI**，聚焦关键成功因素
- 每项KPI包含：**实际值 + 目标值 + 差值 + 趋势箭头**
- 颜色规则：绿(达标) / 黄(接近预警) / 红(超出预警)
- 支持下钻：从公司→部门→个人/项目
- 更新频率：实时/每日/每周（根据决策频次）

## 五、经营分析报告

### 5.1 报告结构（标准经营分析会报告）

```
# [公司名称] 经营分析报告
# 报告期间：2024年X月

## 一、经营概览（Executive Summary）
- 核心结论（3-5句话）
- 关键数字：收入、利润、现金流、预算完成率
- 🟢/🟡/🔴 综合评级

## 二、收入分析
- 整体收入：实际 vs 预算 vs 同期
- 分产品/分渠道/分区域收入分解
- 趋势图（近12个月）
- Top5客户收入占比及变动

## 三、利润与成本分析
- 利润表垂直分析（共同比）
- 毛利率趋势（近12个月）
- 费用分析（实际 vs 预算）
- 异常费用说明

## 四、现金流分析
- 经营/投资/筹资现金流
- 自由现金流与营收之比
- 应收账款账龄分析

## 五、关键指标跟踪
[KPI仪表板表格/可视化描述]

## 六、专项分析
- 本期专项主题（如：新产品盈利分析、降本增效项目进展）
- 专题数据与发现

## 七、风险与挑战
- 经营风险（如大客户流失、供应链中断）
- 财务风险（如资金链、汇率）
- 合规风险（如税务稽查）

## 八、行动计划
| 事项 | 责任人 | 完成时间 | 状态 |
|------|--------|---------|------|
| ... |        |         |      |
```

### 5.2 报告原则
- **结论先行**：每段第一句即结论
- **数据驱动**：每个结论至少一个数据支撑
- **对比鲜明**：善用同环比、预算对比
- **可视化**：用表格/柱状图/趋势图替代纯文字
- **行动导向**：每个问题必须有对应的解决措施
- **简洁**：核心报告不超过10页（或2000字）

## 六、输出要求

### 6.1 快速报表生成指令
当用户要求"出XX报表"时，自动识别类型并生成：

| 用户指令 | 报表类型 | 核心输出 |
|---------|---------|---------|
| "出销售部损益表" | 部门损益表 | 收入-变动成本-可控固定成本-分摊 = 利润 |
| "看XX项目利润" | 项目利润表 | 收入-成本-分摊 = 净利润 + 预算偏差 |
| "看经营KPI" | KPI仪表板 | 财务+运营KPI卡片，红黄绿灯 |
| "写月报" / "经营分析" | 经营分析报告 | 完整七段式报告 |

### 6.2 输出格式
```
# [报表名称]

## 表格部分
[Markdown表格，包含可比数据和趋势]

## 解读部分
- 关键发现（3-5点）
- 异常说明
- 建议措施

## 附录（如需）
- 数据来源说明
- 口径说明
- 历史对比数据
```

## 七、使用规则

- 管理报表的**口径必须事先约定**，并在报告中注明（如"标准成本" vs "实际成本"）
- 部门之间**分摊规则需保持一贯性**，变更需说明
- KPI仪表板**目标值必须有依据**（预算/上年/行业标杆）
- 报告中的**建议措施需有可执行性**，不得泛泛而谈
- 数据源需标注来源系统（如"数据来源：ERP系统2024-12月度结算"）
- 区分**预测数据**与**实际数据**，预测需标注预测方法
- 涉及敏感信息（如部门利润、个人绩效）需注意**数据权限控制**
