# 凭证录入与审核

## 概述

本技能涵盖日常会计凭证的编制、审核与错误检查，是财务核算的基础环节。适用于企业日常经济业务的分录处理、凭证合规性审查及常见错误的自动识别与修正。

## 日常会计分录编制

### 基本要求
- 每笔分录必须有借有贷，借贷金额相等
- 凭证日期按经济业务发生日期填列，不得跨期
- 摘要简明扼要，如实反映经济业务内容
- 附单据必须齐全，金额与凭证一致

### 常见业务分录模板

| 业务类型 | 借方科目 | 贷方科目 | 说明 |
|---------|---------|---------|------|
| 采购原材料 | 原材料 | 银行存款/应付账款 | 价税分离，进项税单独列示 |
| 销售商品 | 银行存款/应收账款 | 主营业务收入 | 同时结转成本：借主营业务成本，贷库存商品 |
| 支付工资 | 应付职工薪酬 | 银行存款 | 先计提后发放 |
| 计提折旧 | 管理费用/制造费用 | 累计折旧 | 按受益部门分配 |
| 报销差旅费 | 管理费用/销售费用 | 银行存款/其他应收款 | 冲抵借支余额 |

### 凭证类型
- **记账凭证**：通用格式，适用于各类业务
- **收款凭证**：专门记录现金/银行存款收入
- **付款凭证**：专门记录现金/银行存款支出
- **转账凭证**：记录不涉及现金/银行存款的转账业务

## 凭证审核要点

### 形式审核
1. **完整性检查**：凭证编号连续、日期完整、附件齐全
2. **签章检查**：制单人、审核人、记账人、出纳签章齐全
3. **科目合规性**：使用一级科目及明细科目是否正确，科目编码是否匹配

### 内容审核
1. **金额稽核**：检查大小写金额是否一致、计算是否准确
2. **借贷平衡**：确保每张凭证借贷方合计金额相等
3. **方向正确**：费用类科目一般在借方，收入类科目一般在贷方
4. **关联校验**：与原始单据逐笔核对，确保业务真实

### 重点检查项目
- 现金/银行凭证的收付方向是否正确
- 往来科目（应收/应付）是否与客户/供应商匹配
- 税金科目计算与申报表是否一致
- 跨期费用的归属期间是否正确

## 常见错误检查

### 金额类错误
- **借贷方金额不相等的凭证**
- **金额错位**：如 1,000 元录入为 100 元
- **红字金额使用不当**：冲销应使用红字（负数）而非反向分录

### 科目类错误
- **科目选用不当**：如将运费计入管理费用而非采购成本
- **明细科目串户**：挂错客户/供应商辅助核算
- **科目方向错误**：将贷方发生额计入借方

### 勾稽关系错误
- 应收账款余额与客户对账单不符
- 存货账面余额与盘点结果差异超出合理范围
- 银行存款余额调节表中的未达账项长期未处理

## 自动化检查脚本

可使用以下查询快速识别异常凭证：

```sql
-- 检查借贷不平凭证
SELECT voucher_id, SUM(debit_amount) AS total_debit,
       SUM(credit_amount) AS total_credit
FROM voucher_detail
GROUP BY voucher_id
HAVING ABS(SUM(debit_amount) - SUM(credit_amount)) > 0.01;
```

```python
# 检查凭证金额异常（示例脚本片段）
def detect_abnormal_vouchers(vouchers: list) -> list:
    """检测金额异常的凭证"""
    anomalies = []
    for v in vouchers:
        # 检查金额是否超过科目历史均值的 5 倍
        if v.amount > v.historical_avg * 5:
            anomalies.append({
                'voucher_id': v.id,
                'reason': f"金额 {v.amount} 超过均值 {v.historical_avg:.2f} 的 5 倍"
            })
    return anomalies
```

## 最佳实践

1. **日清月结**：每日凭证当日整理，每月末及时结账
2. **凭证摘要规范化**：制定摘要模板，避免口语化描述
3. **审核分离**：制单与审核岗位分离，不得自审自核
4. **凭证归档**：按编号顺序装订成册，封面标注起止编号及日期
5. **电子备份**：财务软件中定期备份凭证数据
