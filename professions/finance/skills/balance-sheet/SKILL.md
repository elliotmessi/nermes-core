# 资产负债表分析

使用本技能时，你需要对企业的资产负债表进行**结构分析、科目重分类、期初期末变动分析**，并输出专业的分析报告。

---

## 一、数据准备

### 1.1 获取数据
- 确认数据来源：财务报表原始数据（Excel/CSV）、财务系统导出、企业财报PDF
- 所需字段：报表日期、资产类科目及金额、负债类科目及金额、权益类科目及金额
- 至少需要**两期数据**（本期 vs 上期，或期末 vs 期初）才能做变动分析

### 1.2 数据清洗
- 确认科目名称标准化，同一科目不同期次名称一致
- 处理负值：往来款项借项负值应重分类至对方科目（详见第三节）
- 确认合计行与明细行不重复计算

---

## 二、结构分析

### 2.1 资产结构分析
- **总资产** = 流动资产 + 非流动资产
- **流动资产占比** = 流动资产 / 总资产 × 100%  → 衡量企业资产流动性
- **非流动资产占比** = 非流动资产 / 总资产 × 100%
- 关键子项分析：
  - 货币资金占比 → 支付能力
  - 应收账款占比 → 回款风险
  - 存货占比 → 积压风险
  - 固定资产占比 → 重资产程度

### 2.2 负债结构分析
- **总负债** = 流动负债 + 非流动负债
- **流动负债占比** = 流动负债 / 总负债 × 100%  → 衡量短期偿债压力
- **资产负债率** = 总负债 / 总资产 × 100%  → 杠杆水平
- 关键子项分析：
  - 短期借款占比 → 短期融资依赖度
  - 应付账款占比 → 供应商融资利用
  - 长期借款占比 → 长期资金规划

### 2.3 权益结构分析
- **净资产** = 实收资本 + 资本公积 + 盈余公积 + 未分配利润
- **各权益科目占比** = 各科目 / 净资产 × 100%
- **未分配利润占比** → 反映企业留存收益积累程度

## 三、科目重分类

某些科目在报表中日径可能不准确，需要进行重分类调整：

### 3.1 往来款项重分类
- **应收账款贷方余额**（负值）→ 重分类至「预收账款」或「合同负债」
- **应付账款借方余额**（负值）→ 重分类至「预付账款」
- **其他应收款贷方余额** → 重分类至「其他应付款」
- **其他应付款借方余额** → 重分类至「其他应收款」

### 3.2 一年内到期的非流动负债
- 长期借款中一年内到期部分 → 重分类至「一年内到期的非流动负债」（流动负债）
- 长期应付款、长期应付债券同理

### 3.3 其他常用重分类
- **应收票据**已贴现未到期 → 根据是否终止确认判断是否出表
- **长期股权投资**减值准备 → 确认是否已充分计提
- **使用权资产与租赁负债** → 按新租赁准则确认

---

## 四、期初期末变动分析

### 4.1 绝对变动
- **变动额** = 期末余额 - 期初余额（或本期 - 上期）
- 正数 = 增加，负数 = 减少

### 4.2 相对变动
- **变动率** = 变动额 / 期初余额 × 100%
- 重点关注变动率超过**20%** 或绝对金额影响超过**总资产5%** 的科目

### 4.3 异常变动识别
- 货币资金大幅减少→需关注现金流状况
- 应收账款增幅远大于收入增幅→回款质量下降
- 存货大幅增加且周转率下降→积压风险
- 短期借款替代长期借款→期限结构错配风险
- 未分配利润为负→累计亏损

### 4.4 勾稽关系校验
- **会计恒等式**：资产 = 负债 + 所有者权益（必须相等）
- 期初资产 + 本期变动 = 期末资产
- 净利润（利润表）→ 归入未分配利润（资产负债表）

## 五、输出要求

分析完成后，以结构化 Markdown 输出报告，包含：

```
# 资产负债表分析报告

## 一、总体概况
[公司名称] | [报表日期] | [货币单位]

## 二、资产结构分析
| 项目 | 期末余额 | 占比 | 期初余额 | 占比 | 变动额 | 变动率 |
|------|---------|------|---------|------|-------|-------|
| ...  |         |      |         |      |       |       |

## 三、负债与权益结构分析
[同上表格格式]

## 四、重分类调整说明
[列出所做的重分类调整]

## 五、关键发现与建议
1. 重点关注科目及原因
2. 风险提示
3. 改善建议
```

---

## 六、使用规则

- 科目数据务必**双向校验**（资产=负债+权益）
- 重分类调整必须标注调整依据
- 变动分析需结合业务实质解释，不要仅罗列数字
- 异常识别需要给出明确阈值判断标准
- 报告语言保持专业、中立、客观
