# 审计追踪技能

## 技能描述

审计追踪技能用于企业财务数据的全流程审计轨迹记录与合规性检查。系统自动记录每一笔财务数据从创建、修改到删除的完整操作历史，包括操作人、操作时间、操作IP、原始值和新值等详细信息。支持按时间段、操作类型、用户等维度进行审计日志查询和分析，并内置异常行为检测引擎，可自动识别和告警潜在的财务风险行为。

## 适用场景

- **内部审计**：企业内部审计部门定期审查财务数据变更记录，评估内部控制有效性
- **外部审计配合**：为会计师事务所的年审提供完整的电子审计轨迹和证据链
- **合规检查**：满足 SOX、IPO 等监管要求，证明财务数据的完整性和不可篡改性
- **异常行为排查**：发现账务数据异常变动时，快速定位责任人并回溯操作过程

## 输入格式

```json
{
  "audit_query": {
    "object_type": "voucher",
    "object_id": "记-2026-05-23-001",
    "start_time": "2026-05-23 00:00:00",
    "end_time": "2026-05-23 23:59:59",
    "operations": ["create", "update", "delete", "audit"],
    "user_id": ""
  }
}
```

## 注意事项

- 审计日志应存储在独立的、只能追加写入的存储介质中，防止被非法篡改
- 关键财务数据的修改权限应严格管控，建议执行双人复核制度
- 审计日志保留年限不得少于法律法规要求的最低期限（一般不少于10年）
- 对于频繁修改凭证、越权操作等异常行为，系统应实时触发告警通知
- 定期（建议每季度）对审计日志进行完整性校验，确保日志未被篡改或删除
