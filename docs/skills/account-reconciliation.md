# 对账处理技能

## 技能描述

对账处理技能用于自动化执行企业内部账户及内外账户之间的数据核对与差异分析。支持银行对账单与账面余额的逐笔勾对、往来款项的客户/供应商对账、总账与明细账的账账核对，以及内部关联交易的对账抵消。系统通过智能匹配算法自动识别匹配项，对未达账项和差异项进行高亮标记，并生成差异调节表。

## 适用场景

- **银行存款对账**：每月将企业银行日记账与银行对账单进行逐笔核对，生成银行存款余额调节表
- **往来款项对账**：定期与客户、供应商核对应收应付余额，发现差异后自动推送差异报告
- **内部往来对账**：集团内部各子公司之间关联交易的核对与抵消
- **科目余额对账**：总账科目余额与明细账汇总金额的一致性校验

## 输入格式

```json
{
  "reconciliation_type": "bank",
  "account_id": "1002-001",
  "period": "2026-04",
  "company_side": [
    {"date": "2026-04-15", "voucher_id": "银付-001", "description": "支付货款", "amount": 50000.00, "balance": 250000.00}
  ],
  "bank_side": [
    {"date": "2026-04-15", "bank_seq": "BANK-20260415001", "description": "转账支出", "amount": 50000.00, "balance": 245000.00}
  ]
}
```

```
减：银行已付企业未付    5,000.00
调节后余额：330,000.00

银行对账单余额：328,000.00

差异项明细：
  ✓ 已匹配：142 笔
  ✗ 未匹配：3 笔（企业2笔，银行1笔）
```

## 注意事项

- 对账前确保所有凭证已全部入账，避免因凭证遗漏导致虚假差异
- 对于长期未达账项（超过3个月），应查明原因并及时处理
- 银行对账建议每月进行一次，往来对账建议每季度进行一次
- 对账差异调整需保留完整的审计轨迹，注明调整原因和审批人
- 自动化匹配率一般可达90%-95%，剩余未匹配项需人工判断
